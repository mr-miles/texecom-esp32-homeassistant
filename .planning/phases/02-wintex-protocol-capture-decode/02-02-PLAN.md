**Required sections**:
1. **Sources** — list each capture file used, the date/scenario it was recorded under (per the user-instruction block in 02-01-SUMMARY), and the community references consulted
2. **Framing** — start byte (if any), length encoding, checksum algorithm + position, terminator (if any). This is THE most important section — get framing right or every payload decode is wrong

   - Heartbeat / keepalive
   - Zone state (each zone's open/closed/tamper state)

   - Disarm command + ack

   - Tamper event

**Citation discipline**: every message definition cites the capture file and byte offset where it was first observed. Future contributors / Phase 3 / Phase 5 readers can re-validate every claim.

**`wintex_events.h`** — define a tagged-union (`std::variant` if available, else discriminated struct) of event types corresponding to Section 4 of `.planning/protocol.md`:

```cpp
struct ZoneStateEvent { uint8_t zone; enum class State : uint8_t { Closed, Open, Tamper, Masked } state; };
struct AreaStateEvent { uint8_t area; enum class State : uint8_t { Disarmed, ArmedAway, ArmedStay, Entry, Exit, Alarm } state; };
struct ArmAckEvent    { uint8_t area; bool success; };
struct DisarmAckEvent { uint8_t area; bool success; };
struct AlarmEvent     { uint8_t area; uint8_t zone; };  // zone may be 0 if area-wide
struct TamperEvent    { uint8_t zone; bool tampered; };
struct BatteryEvent   { uint16_t millivolts; bool low; };
struct MainsEvent     { bool present; };
struct HeartbeatEvent { /* maybe a sequence number */ };
// ... etc

using WintexEvent = std::variant<ZoneStateEvent, AreaStateEvent, ArmAckEvent, DisarmAckEvent, AlarmEvent, TamperEvent, BatteryEvent, MainsEvent, HeartbeatEvent>;
```
(If targeting C++17 — std::variant is available. ESPHome supports C++17 on ESP32.)

**`wintex_decoder.h`** — class:
```cpp
class WintexDecoder {
 public:
  WintexDecoder(const PanelModel *model);
  void on_byte(uint8_t b);     // streaming input
  bool has_event() const;
  WintexEvent pop_event();
  // Diagnostic counters for dump_config/tests
  size_t bad_checksum_count() const;
  size_t unknown_message_count() const;
};
```
Internal state machine consumes bytes per the framing rules in `.planning/protocol.md`. When a complete message is parsed, it dispatches via PanelModel's message-ID table to construct a `WintexEvent` and queues it. Bad checksums increment a counter and are dropped; unknown message IDs increment a counter and are dropped (with optional debug logging).

**`panel_model.h`** — extend the abstract base:
```cpp
struct WintexMessageSpec {
  uint8_t id;
  uint8_t expected_length;  // or 0xFF for variable
  const char *name;
};

class PanelModel {
 public:
  virtual ~PanelModel() = default;
  virtual const char *name() const = 0;
  virtual uint8_t zone_count() const = 0;
  virtual uint8_t area_count() const = 0;
  virtual const WintexMessageSpec *message_specs() const = 0;  // null-terminated
};
```

**`panel_model_premier24.{h,cpp}`** — populate with the exact values from `.planning/protocol.md` Section 6. Verify and correct the placeholder zone_count/area_count if the captures show different numbers.

**Decoder is host-testable**: no AsyncTCP, no ESPHome runtime, no UART. Tests will be added in Plan 02-03.
